# Security Policy

## Reporting a Vulnerability

Please **do not** open a public issue for security vulnerabilities.

Instead, use [GitHub's private vulnerability reporting](https://github.com/bradleyoesch/planderson/security/advisories/new) to submit a report confidentially. You'll receive a response once the report has been reviewed.

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)
